◆ run()

const pub pub fn run ( options:ExampleOptions ) -> Result<()>

Run the on-premise data-file-update example.

Illustrates the four ways an on-premise data file is kept current: update-on-startup, the file-system watcher, remote-update polling, and a programmatic (on-demand) update. It runs the programmatic path for real against the supplied file (a safe no-network reload from disk) and documents the configuration for the other three.

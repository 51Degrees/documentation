# Pattern Lab Project

A starting point for new web projects using using [atomic design](http://bradfrost.com/blog/post/atomic-web-design/) methodology and Pattern Lab.

**Primary Purpose**: This pattern library is primarily used to generate CSS for the [51Degrees documentation](https://github.com/51degrees/documentation) website. The main output is the `docs-main.css` and `docs-main.min.css` files that provide styling for the documentation pages.

## Features

- [Pattern Lab](http://patternlab.io/) ([Twig](https://github.com/pattern-lab/edition-php-twig-standard) and [PHPTemplate](https://github.com/aleksip/edition-php-tpl) editions)
- [Gulp](http://gulpjs.com/)
- [Sass](http://sass-lang.com/)
- [Browsersync](https://browsersync.io/)

## Getting started

### Prerequisites

[npm](https://nodejs.org/) and [Composer](https://getcomposer.org/) installed.

### Installing

```bash
npm install
```

```bash
if [ ! -d pattern-lab ]; then composer create-project -n pattern-lab/edition-twig-standard pattern-lab && cd pattern-lab && composer require aleksip/plugin-data-transform && php core/console -c --set sourceDir=../source && php core/console -c --set twigAutoescape=0 && php core/console --generate && cd ..; fi
```

### Running

```bash
gulp
```

## CSS Generation for Documentation

This pattern library generates CSS files specifically for the [51Degrees documentation](https://github.com/51degrees/documentation) website. The build process compiles SCSS files into optimized CSS output.

### Docker Build (Recommended)

To generate CSS files using Docker:

```bash
docker build --output=build .
```

This command will:
- Compile SCSS files to CSS
- Add vendor prefixes
- Minify CSS output
- Generate both `docs-main.css` and `docs-main.min.css` in the `build/` directory

The generated CSS files need to be manually copied to the [documentation](https://github.com/51degrees/documentation) source directory:
- `/documentation/docs/`

The `/documentation/<version>/` directories are generated by Doxygen and will automatically receive the CSS files during the documentation build process.

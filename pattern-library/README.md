# 51Degrees documentation pattern library

The design system for the [51Degrees documentation](https://github.com/51degrees/documentation)
site, organised with [atomic design](http://bradfrost.com/blog/post/atomic-web-design/)
methodology.

**Primary purpose:** generate the CSS for the documentation site. The main
output is `docs-main.css` / `docs-main.min.css`, which style the documentation
pages.

The design system is previewed with **Storybook**. (The previous Pattern Lab /
Twig / PHP preview has been retired — it was unmaintained and carried a large
amount of dependency-security debt; Storybook replaces it with no PHP/composer
toolchain.)

## Features

- Atomic-design SCSS in `source/sass` (`00-abstract` … `05-pages`)
- [Gulp](http://gulpjs.com/) + [Dart Sass](http://sass-lang.com/) — compiles SCSS → CSS
- [Storybook](https://storybook.js.org/) (`@storybook/html-vite`) — component preview

## Prerequisites

[Node.js](https://nodejs.org/) 20+ and npm. (No PHP or Composer required.)

## Getting started

```bash
npm install
```

### Preview the design system (Storybook)

```bash
npm run storybook
```

This compiles the SCSS (`gulp sass`) and starts Storybook on
<http://localhost:6006>. Stories live in `stories/` — each is plain HTML that
uses the design-system classes against the compiled `docs-main.css`. To add a
component, create `stories/<Name>.stories.js`; wrap documentation content in
`.l-docs-index .g-docs__primary` to pick up the documentation layout styles.

Build a static Storybook site:

```bash
npm run build-storybook   # -> storybook-static/
```

## CSS generation for the documentation site

The documentation build runs `gulp sass-change` (compile → autoprefix →
minify), producing `docs-main.css` and `docs-main.min.css`.

### Docker build

```bash
docker build --output=build .
```

Compiles the SCSS, adds vendor prefixes, minifies, and writes the CSS to
`build/`. The documentation build (`ci/build-pattern-library.ps1`) copies the
`docs-*.css` output into `/documentation/docs/`; the per-version directories
are generated by Doxygen and receive the CSS during the documentation build.

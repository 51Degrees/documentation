◆ display_values()

pub fn display_values ( ip:&dyn IpIntelligenceData) -> Vec< DisplayValue >

Gather the displayed IP-intelligence properties from the result, read through the strongly-typed accessors.

The order is network registration first, then the textual location, the coordinates and accuracy, then the network flags and the weighted country-code distributions.

Plain properties (strings, the [f32] coordinates, the [i32] radii and the [bool] flags) resolve to a single value. The weighted properties at the end (CountryCodesGeographical, CountryCodesPopulation, Mcc) resolve to a weighted list, shown as value (weight) pairs.

◆ display_weighted()

fn display_weighted ( label:& str,
property:&AspectPropertyValue< Vec< WeightedValue< String > > >  ) -> DisplayValue

Reduce a weighted [AspectPropertyValue] to its display value, rendering each candidate as value (weight) ordered high weighting first and joined with commas.

A no-value carries the engine's explanation.

This is how the genuinely weighted properties (the country-code distributions and Mcc) are shown: a single IP can overlap several countries, so the value is a weighted list rather than one definite value.

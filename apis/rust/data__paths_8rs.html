◆ IpiTier

enum IpiTier

Which tier of IP Intelligence data file to resolve.

The default, [IpiTier::BestAvailable], picks the best loadable tier: Enterprise when the configured share is reachable, otherwise ASN. A caller can force a specific tier when an example needs it, for example to exercise the Enterprise-only properties.

◆ ipi_data_path()

pub fn ipi_data_path ( tier:IpiTier ) -> Option< PathBuf >

Resolve the path to an on-premise IP Intelligence data file at the requested tier.

[IPI_PATH_ENV_VAR] always wins when set to a non-blank path, regardless of tier, so a caller can point every example at one explicit file. Otherwise the tier decides:

  • [IpiTier::BestAvailable] returns the latest Enterprise file from the configured share if it is reachable, and otherwise the ASN file.
  • [IpiTier::Enterprise] returns the latest Enterprise file, or None when the share is unreachable.
  • [IpiTier::Asn] returns the ASN file located in a sibling ip-intelligence-cxx checkout.

Returns None when no file for the requested tier can be located.

◆ latest_enterprise_ipi()

pub fn latest_enterprise_ipi ( ) -> Option< PathBuf >

Locate the latest dated Enterprise IP Intelligence file on the configured share ([IPI_ENTERPRISE_SHARE_ENV_VAR]), or None when the share is unset or not reachable.

The share is laid out as <root>/YYYY/MM/DD/<file>. The most recent dated folder that actually contains the file is chosen, comparing folder names lexicographically (zero-padded YYYY, MM and DD sort the same as by date). When the variable is unset, or any I/O error occurs (the common case when the share is not mounted), the result is None, so the resolver never blocks or panics.

pub const IPI_ASN_RELATIVE_PATH

The ASN IP Intelligence data file checked into the ip-intelligence-data repository (resolved through a sibling ip-intelligence-cxx checkout).

As the data file committed to the repository it loads against the current native library and is the convenient default for quick tests and CI. A full IP Intelligence data file is obtained from the data repository (see its README) when richer properties are needed.

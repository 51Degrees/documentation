◆ build_app()

pub struct Options build_app ( data_file:&std::path::Path ) -> anyhow::Result<(Router, Vec< String >)>

Build the on-premise web application router plus the data-file warnings.

The engine is built once so its data-file warnings (free tier, and over 30 days old) can be read with [examples_shared::check_data_file] and shown on the page, and the very same engine instance is then used as an application element of the web pipeline. Usage sharing is added first (REQUIRED for a web example).

◆ get()

async fn get ( app:Router ,
uri:& str ) -> Response

Drive one GET request through a clone of the app and return the response.

A clone is used (cloning a Router is cheap, sharing the same Arc elements) so the owning app the caller holds outlives the request. The usage-sharing element wraps a blocking reqwest client whose backing runtime must not be dropped from inside an async context, so keeping the owning app in the synchronous test body lets its final drop happen outside the runtime.

Build the app for the test, skipping when no data file is available.

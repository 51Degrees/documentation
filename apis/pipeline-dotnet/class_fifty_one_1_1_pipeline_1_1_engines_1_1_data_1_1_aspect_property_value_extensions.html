FiftyOne.Pipeline.Engines.Data.AspectPropertyValueExtensions Class Reference

Detailed Description

Safe accessors for IAspectPropertyValue<T> that return a fallback instead of throwing when a value cannot be provided.

There are two distinct failure modes when reading an aspect property, and callers commonly need to treat both as "value unknown":

  1. The property reference resolves but has no value for this evidence. IAspectPropertyValue.HasValue is false and reading IAspectPropertyValue<T>.Value throws Exceptions.NoValueException.
  2. The property reference itself cannot be resolved, for example because an upstream cloud request failed or the data file does not include the property. The typed property accessor (such as a device-data HardwareName getter) throws PropertyMissingException before any IAspectPropertyValue<T> instance exists, so extensions on the property value cannot guard it. The selector overloads below take the accessor as a function and guard the whole read.

FiftyOne.Pipeline.Engines.Services.MissingPropertyServiceOnPremise Class Reference

Detailed Description

On-premise missing-property service.

Inherits all behaviour from MissingPropertyService unchanged — on-premise engines do not depend on the per-request cloud state handled by MissingPropertyServiceCloud, so the base heuristics already report the correct reason.

This type exists so that callers make a deliberate cloud-vs-on-premise choice: cloud aspect-engine builders should wire MissingPropertyServiceCloud, on-premise builders should wire this service. The deprecated MissingPropertyService.Instance singleton remains only to surface that choice.
